## 二、关键发现：所有方案共享三个限制

### 限制1: 几乎全是白盒

| 方法 | 需要 |
|------|------|
| Semantic Entropy / Bayesian SE / SHADE | 多次采样(5-20次) 或 logprobs |
| LSD | 所有层的hidden states |
| Graph Signal Processing | 注意力矩阵 |
| Entropix / Klarity | 每步token logits |
| Chatsparent | hidden states + 注意力权重 |

**只有 Entropy Sentinel 能在 API-only 条件下工作**（需要 `top_logprobs` 参数，不是所有API都支持）。

### 限制2: 全部检测"生成内容"，不检测"agent行为"

所有方案都在分析模型**说了什么**（输出的语义/熵/轨迹），没有一个分析 agent **做了什么**（工具调用序列是否异常、决策模式是否变化）。

一个陷入死循环的 agent 可能在每次循环中输出语义上完全合理的文本——语义熵检测不到，因为每次单看都对。

### 限制3: 都是事后或半实时

除Entropix(需要logits)和Klarity(需要logits+外挂LLM判断)外，都需要多轮采样或批处理。没有方案能在**每次工具调用前、200ms内、零API成本**完成检测。

## 三、CLS符号动力学的差异定位

### 不是语义检测，是行为检测

```
其他所有方案:
  模型输出文本 → 分析语义/熵/谱 → 判断这句是否幻觉

CLS符号动力学:
  工具调用序列 → 映射为离散符号 → 分析符号流的统计特征 → 判断agent是否偏离正常
```

一个直观例子：

```
Agent在执行任务:
  Step 1: Bash成功 → Bash成功 → Bash成功 → Bash成功 → ...
  
语义熵方案: 每次Bash输出都合理，检测不到问题
CLS方案: 符号流[h,h,h,h,h,h...] → Shannon熵≈0 → 检测到死循环
```

### 核心差异表

| 维度 | 所有其他方案 | CLS符号动力学 |
|------|------------|-------------|
| 监控对象 | 模型输出的**文本内容** | Agent的**工具调用行为** |
| 数据源 | 需要模型内部状态 (logits/hidden/attention) | 工具调用日志+hook裁决 (API无关) |
| 数学工具 | 连续概率/几何/谱分析 | 离散符号动力学 (熵/拓扑熵/禁止词/子移位) |
| 检测类型 | 逐句事实幻觉检测 | **跨时间行为模式异常** (死循环/相变/漂移) |
| 实时性 | 多采样→秒级；或需logits访问 | ~200ms，纯计算 |
| API成本 | 多数需要多采样或多模型 | 零 |
| 理论框架 | 信息论/微分几何/谱图论 | **符号动力学 + 子移位 + 禁止词** |

## 四、CLS方案在学术文献中的独特位置

### 已有人做过的（≠CLS）
- Shannon熵检测幻觉 ✓（Farquhar 2024）
- 谱半径检测异常 ✓（Noël 2025）
- 禁止词/模式过滤 ✓（jailbreak检测文献）

### 没有人做过的（=CLS独有）

1. **把agent工具调用序列视为离散符号动力系统**
   - LSD用几何轨迹分析hidden states
   - CLS用符号序列分析tool calls
   - → 完全正交的方法论

2. **用子移位(SFT)定义agent的"合法行为空间"**
   - 禁止词 = 不可出现的工具调用模式
   - 这是一个在LLM监控领域从未被提出的框架
   - 最近的jailbreak分类论文(2510.15476)接近但未引入SFT概念

3. **三维度组合检测(熵+拓扑熵+禁止词)**
   - 熵检测行为单调性
   - 拓扑熵检测行为开放性
   - 禁止词检测已知危险模式
   - → 三者组合覆盖"太死/太活/已知坏"三个维度

4. **跨域独立监控**
   - 8个域各有自己的字母表和转移矩阵
   - 一个域异常不影响其他域
   - → 不是全局阈值，是域级健康评分

5. **与语义验证的正交性**
   - 符号动力学 = 统计健康检查 (便宜、快、行为级)
   - 双AI闸门 = 语义验证 (贵、慢、内容级)
   - → 两者不是竞争关系，两层各自捕获对方遗漏的信号

## 五、如果要强调CLS的独特性，重点提这几点

### 1. 方法论独特
在LLM监控领域引入**符号动力学的完整框架**（有限字母表→观察序列→熵率+拓扑熵+禁止词→稳定性评分），目前没有论文或开源项目这样做。

### 2. "行为"而非"内容"
所有主流方案监控模型**说什么**。CLS监控agent**做什么**。这两类信号正交——一个agent可能说了完全合理的句子，但整体行为模式已经异常。

### 3. 黑盒兼容
不需要模型内部状态。对API调用的模型同样有效。这是绝大多数白盒方案做不到的。

### 4. 实时+零成本
200ms/次，零API调用。可以嵌入PreToolUse hook做每次工具调用前的实时检查。

### 5. 子移位(SFT)的工程化
把"合法行为空间"定义为避开禁止符号序列的集合，这是理论计算机科学中成熟的形式语言概念首次被用于LLM agent的行为约束。
